# UI Notes: FOR-81

## Applicability

This story owns only the frontend shell/skeleton.

## Requirements

- Provide a basic route and layout placeholder.
- Prepare a feature-folder structure only if it helps future stories.
- Keep responsive layout assumptions minimal.
- Keep API access behind a client boundary.

## Constraints

- No product dashboard.
- No fake body/training/nutrition data.
- No hardcoded user-specific data.
- No domain calculations in components.

## Future-Proofing

The skeleton should make later UI stories easy to add without requiring a full restructure, but it must not pretend to implement the future design system.
