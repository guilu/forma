# FOR-85: Configure code formatting and linting baseline

Jira: https://dbhlab.atlassian.net/browse/FOR-85
Story points: 2
Epic: FOR-79 Project Bootstrap

## Goal

Configure the initial code formatting and linting baseline for FORMA.

## Business Value

Keeps code generated by humans and agents consistent, reducing review noise and style drift.

## Scope

Define the first formatting/linting baseline.

The baseline should include:

- Backend formatting/linting rules.
- Frontend formatting/linting rules.
- Commands to check formatting.
- Commands to apply formatting where appropriate.
- CI integration where practical.
- Minimal editor-agnostic configuration.

## Architecture Notes

- Align with `docs/coding-standards.md`.
- Prefer standard tooling over custom scripts.
- Keep rules strict enough to prevent chaos, not strict enough to become a hobby.
- Avoid large unrelated formatting churn beyond baseline setup.

## Acceptance Criteria

- Backend formatting/linting command exists.
- Frontend formatting/linting command exists.
- CI can run checks where practical.
- Documentation explains how to run checks locally.
- No large unrelated formatting churn is introduced beyond baseline setup.

## Out of Scope

- Reformatting unrelated future code.
- Adding custom style rules without clear benefit.
- IDE-specific setup as the only supported workflow.

## Definition of Done

- Formatting/linting configuration committed through PR.
- Commands verified locally.
- CI integration added or documented as follow-up.
- Documentation updated.
